## STEP 1: INITIALIZE GIT & PUSH to GitHub
# Run these in PowerShell from your project root (hatobot_v5/hatobot/)

git config --global user.email "your-email@gmail.com"
git config --global user.name "Your Name"

git init
git add .
git commit -m "🚀 HatoBot v5 - Ready for Vercel deployment with FastAPI backend + React admin"
git branch -M main
git remote add origin https://github.com/YOUR_USERNAME/hatobot.git
git push -u origin main

# ✅ On first push, GitHub will ask for authentication token
# Create one at: https://github.com/settings/tokens (classic token, repo scope)
# Then paste when prompted

## STEP 2: INSTALL VERCEL CLI & SET UP
npm install -g vercel

# ✅ First time setup (links your Vercel account)
vercel login

---

## STEP 3: DEPLOY TO VERCEL (PRODUCTION)
# From your project root: hatobot_v5/hatobot/

vercel --prod

# ✅ When prompted:
# - Set project name: hatobot
# - Select existing project (if available) or create new
# - Framework: Other
# - Root directory: .
# - DO NOT modify build/output settings (vercel.json handles it)

## TROUBLESHOOTING

❌ Build fails:
-> Check admin/package.json scripts (dev, build, preview)
-> Ensure Vite is configured correctly
-> Run: npm install && npm run build (in admin folder)

❌ Python errors:
-> Verify api/requirements.txt has all dependencies
-> Check api/index.py for syntax errors
-> Ensure FastAPI exports 'app' variable

❌ Webhook not working:
-> Verify VERIFY_TOKEN matches between Vercel env and Facebook config
-> Check vercel.json routing rules

❌ 404 on frontend routes:
-> Ensure vercel.json has SPA fallback: "src": "/(.*)", "dest": "admin/dist/index.html"

❌ CORS errors:
-> Already handled in api/index.py (CORSMiddleware with allow_origins=["*"])
